Pi 400 Setup & Deploy

10:45 – 11:30
✓ Section unlocked — time to build your machine
Before you start: This is your machine for the whole camp. Take your time, follow each step carefully, and call your partner over if something looks wrong. One typo in a command can break everything — read twice, type once.
01
Plug in your Pi 400 — power cable, HDMI to monitor (use the port closest to the USB ports), ethernet cable into the switch. Give the ethernet 30–60 seconds to connect after plugging in.
02
First boot wizard — set your country, language, timezone. Create your username (your first name) and password: Password01. Skip the software update prompt.
03
Open Terminal — click the black monitor icon in the taskbar. Your prompt will look like yourname@raspberrypi:~ $ — that's normal.

Command reference

step 1 — install ollama
# Download and install Ollama curl -fsSL https://ollama.com/install.sh | sh
Expected warning: You'll see "no NVIDIA/AMD GPU detected, running in CPU-only mode" — this is completely normal on the Pi 400. Ignore it and keep going.
step 2 — pull gemma 2b model (~2 min)
# Download the Gemma 2B model to your Pi ollama pull gemma:2b
step 3 — run your model
# Start Gemma 2B — wait ~10 seconds for >>> prompt ollama run gemma:2b
First prompt to try: "What is artificial intelligence in one sentence?" — watch how it types one word at a time. That's the model predicting the next token in real time.
ollama commands (type inside the >>> prompt)
# Exit Ollama — ALWAYS use /bye, not exit or quit /bye # Clear conversation history /clear # Show available commands /help

Open Exploration & Wrap Up

2:15 – 4:00
✓ Final block unlocked — let's see what you can break

Two rounds. First you break it. Then you fix it. Both happen on the camp server — use your account in Chromium.

Round 1 — Stump the AI (2:20 – 2:35)

01
Goal: Find something that breaks, confuses, or produces a hilariously wrong response. Try both your Pi AND the camp server with the same prompt.
02
Starter ideas: Ask about recent news. Ask it to do complex math. Ask something hyper-local (best restaurant in Alexandria MN). Ask a trick question. Ask it to predict the future.
03
Save your best failure — screenshot or write it down. You'll share it with the class.

Round 2 — Fix it with a System Prompt (2:35 – 3:10)

What is a system prompt? Instructions you give the AI before the conversation starts. It shapes its personality, what it knows, and how it responds. It's the difference between a generic AI and one built for a specific purpose.
01
Build your AI personality on the camp server. Use the system prompt field in the chat settings.
02
Your system prompt should define: who the AI is, what it specializes in, how it talks, what it will/won't do, any special rules.
03
Test it with at least 5 prompts — iterate until it behaves the way you want. Then be ready to demo it.

Inspiration: A sarcastic AI that only speaks in movie quotes. A gaming coach for your favorite game. A pirate who happens to be a cybersecurity expert. A hype man that turns everything into a motivational speech. A brutally honest code reviewer. An AI that explains everything using food analogies.
